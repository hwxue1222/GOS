# 页面设计规范：公司秘书服务「服务详情页」统一布局/组件规范（基于 Change of Director 详情页）

> 目标：在所有“服务详情页”复用同一套信息架构与组件库，仅在“变更字段、签署人、文档类型、主操作”上做差异化。

## 1) 适用范围（服务详情页）
- 适用路由形态：`/corporate-secretary/applications/{serviceType}/[requestId]`
- 典型服务类型来源：
  - 董事变更（Change of Director）
  - 公司信息变更（company update：公司名 / FYE / 地址 / 业务活动 / 秘书 / 股本 等）

## 2) Layout（桌面优先）
- **栅格**：12 栅格 CSS Grid；最大内容宽 1200–1360px，居中。
- **分栏**：左 7（信息内容）/ 右 5（状态+操作+签署/文档）。
- **节奏**：卡片间距 16px；卡片内边距 16–20px；表格行高 44px。
- **粘性**：右侧关键操作卡可 `sticky top: 88px`。
- **响应式**：
  - ≥1024px：双栏；<1024px：单栏堆叠顺序为：概览 → 操作 → 签署&文档 → 变更明细 → 时间线。

## 3) Meta Information（统一规则）
- Title：`{服务名} - 申请详情 - {申请编号}`
- Description：`查看{服务名}申请状态、签署进度与相关文档。`
- OpenGraph：title 同上；type=website。

## 4) Global Styles（与现有系统一致）
- 背景：#F7F8FA；卡片：#FFF；边框 #E6E8EC；圆角 12px；阴影极弱。
- 字体：页标题 18–20 semibold；卡片标题 14–16 semibold；正文 14；辅助 12（#6B7280）。
- 状态：统一 Badge（Success/Warning/Neutral/Danger），避免服务自定义颜色。
- 按钮：Primary/Secondary/Tertiary；危险操作 Destructive。

## 5) 统一信息架构（所有服务详情页必须一致）
### 5.1 顶部标题区（Header）
- 面包屑：公司 / 公司秘书服务 / 申请中心 / 详情
- 主标题：`{服务名}申请详情`
- 右侧紧凑信息：申请编号（可复制）、最后更新时间
- 状态 Badge：Draft / Pending signatures / Pending review / Need more info / Approved / Rejected（与现有枚举保持一致）

### 5.2 左列（信息内容区）
1) **概览卡 SummaryCard（Definition List 两列）**
- 固定字段（所有服务一致）：公司名称、服务类型、提交人、提交时间、当前处理节点/状态（如有）。
- 可选字段（按服务）：Effective date、生效范围、备注。

2) **变更明细卡 ChangeDetailsCard（分组 + 对齐展示）**
- 分组方式：按“变更类别”Accordion；默认展开 1–2 个最关键分组。
- 展示规则：优先“旧值 → 新值”；无旧值则展示新值。

3) **时间线 ActivityTimeline（统一事件模型）**
- 事件：提交、发送邀请、每次签署、文档生成、审核动作、退回原因（若有）。

### 5.3 右列（状态与操作区）
1) **关键操作卡 ActionCard（单一主操作 + 少量次操作）**
- Primary：与当前状态匹配（示例：提醒签署 / 重新提交 / 下载汇总）。
- Secondary：查看历史/导出（仅在现有系统已有时展示）。
- 操作反馈：展示最近一次操作时间与结果（短文本）。

2) **签署与文档整合卡 UnifiedAssetsCard（强制统一）**
- 推荐：Tabs + Table（Tab1=签署，Tab2=文档）保证信息密度与高度稳定。

**签署表 SignaturesTable（若该服务需要签署）**
- 列：签署人｜角色/身份｜状态（Badge）｜发起/截止时间｜完成时间｜操作（查看/提醒/进入签署）
- 空态：显示“No signatures required”或“暂无签署请求”。

**文档表 DocumentsTable（所有服务保留）**
- 列：文档名称｜类型｜状态（Badge）｜生成时间｜操作（预览/下载）

3) **辅助信息卡 HelpCard（可选）**
- 负责人/客服联系方式、FAQ 链接（仅当系统已有时展示）。

## 6) 组件级规范（用于复用与一致性）
- StatusBadge：只读、可复用映射（不要在不同服务里改文案风格）。
- KeyValueList：左对齐 label、右对齐 value；空值用 “—”。
- AccordionGroup：标题含“分组名 + 条目数”；支持默认展开配置。
- DataTable：行 hover 底色 #F3F4F6；操作按钮 icon+文字（避免纯图标）。
- PreviewModal/Drawer：用于“文档预览、签署详情”，关闭后返回当前位置。

## 7) 差异点清单（其它服务详情页如何变化）
> 差异只允许发生在「字段/分组」「签署人逻辑」「文档类型」「主操作文案/可用性」四类。

| 服务类型 | 左列-变更明细分组（示例） | 右列-签署差异 | 右列-文档差异 | 主操作差异（随状态） |
|---|---|---|---|---|
| 董事变更 | Add directors / Remove directors / Effective date | 签署人=现任董事 + 新增董事（Consent） | Resolution 1份 + Consent N份 | Pending signatures：提醒签署；Approved：下载汇总 |
| 公司名变更 | Company name（旧→新）/ 生效日 | 通常与 company update 规则一致（若需要签署则走同表） | 公司名相关决议/表格（按后端生成） | Pending review：提示等待；Need more info：重新提交 |
| FYE 变更 | Financial year end（旧→新）/ 生效日 | 同上 | FYE 相关文档（按后端生成） | 同上 |
| 地址变更 | Registered address（旧→新）/ 生效日 | 同上 | 地址变更相关文档（按后端生成） | 同上 |
| 业务活动变更 | SSIC / Business activity（旧→新） | 同上 | 业务活动相关文档（按后端生成） | 同上 |
| 秘书变更 | Secretary（旧→新）/ 生效日 | 同上 | 秘书变更相关文档（按后端生成） | 同上 |
| 股本变更 | Share capital（旧→新）/ 变更项明细 | 同上 | 股本相关文档（按后端生成） | 同上 |

## 8) 统一验收标准（用于评审）
- 任一服务详情页首屏均可看到：Header + SummaryCard + ActionCard + UnifiedAssetsCard。
- 任一服务的“签署/文档”入口都在同一张卡（Tab 切换），不允许散落到多个区域。
- 变更明细必须分组、字段对齐；长内容默认折叠，避免首屏过长。
